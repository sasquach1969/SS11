01) To Autorun a CLI command at login (not boot)
   A) Go to /home/j.mander/.config/autostart (replace j.mander with your user name)
   B) Create a file named (whatever).desktop (must end in .desktop)
   C) The following is the format for the file.

   [Desktop Entry]
   Type=Application
   Exec=/usr/bin/firefox
   Name=Firefox

   Of course, the example will load Firefox on login. Customize the path
   and name to the application you want to launch at login.
   ***NOTE - with XFCE, there is a much easier way.***
   See Start>Settings>Session and Startup>Application Autostart.
   These isems are stored in the root directory: /etc/xdg/autostart/

03) Unit files are loaded from two locations. From lowest to highest precedence they are:
   /usr/lib/systemd/system/ : units provided by installed packages.
   /etc/systemd/system/ : units installed by the system administrator.
   Use systemctl enable and the program name to create a link that allows a file to load on startup.

04) Power Manager is counterproductive for wifi connections.
   To stop the constant reconnecting, and restore full power to the adaptor:
   Tadd (copy and paste) as a section to /etc/NetworkManager.conf...
   [connection]
   # Values are 0 (use default), 1 (ignore/don't touch), 2 (disable) or 3 (enable).
   wifi.powersave = 2
   After changes are made NM will need to be restarted to take effect
   Code:
   sudo systemctl restart NetworkManager (or better yet,reboot)
